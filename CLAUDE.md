# Work Portfolio - Project Instructions

## Project Context

This repository serves as a professional work portfolio showcasing various projects, proof-of-concepts, and development explorations.

**Primary Purpose**: Demonstrate technical expertise, innovative solutions, and development capabilities

**Audience**: Potential employers, collaborators, and the developer community

## Project Guidelines

### Code Quality Standards

- **Production-Ready Code**: All code should be clean, well-documented, and production-quality
- **Best Practices**: Follow industry best practices and modern development patterns
- **Documentation**: Comprehensive README files for each project with clear examples
- **Testing**: Include tests where applicable to demonstrate quality assurance practices

### Portfolio-Specific Requirements

1. **Showcase Value**: Every project should clearly demonstrate specific skills or techniques
2. **Professional Presentation**: Code should be exemplary and well-organized
3. **Clear Documentation**: Each project must have:
   - Clear purpose and objectives
   - Installation and setup instructions
   - Usage examples
   - Technical highlights
4. **Live Examples**: Where possible, include working examples or demos

### Project Structure

```
work-portfolio/
├── CLAUDE.md           # This file - project-specific instructions
├── README.md           # Main portfolio overview
├── poc/                # Proof-of-concept projects
│   └── project-name/

│       ├── src/
│       ├── docs/
│       └── examples/
├── projects/           # Complete projects
└── experiments/        # Experimental work
```

### Development Workflow

**Branching Strategy**:
- `main`: Production-ready, polished work
- `develop`: Active development
- `feature/*`: New features or projects

**Commit Standards**:
- Clear, descriptive commit messages
- Atomic commits focused on single changes
- Professional commit history suitable for portfolio review

**Code Review Focus**:
- Code clarity and readability
- Documentation completeness
- Example quality and usefulness
- Performance and best practices

### Technology Preferences

- **Modern Stack**: Use current, industry-relevant technologies
- **TypeScript**: Prefer TypeScript for JavaScript projects
- **Testing**: Include unit tests and integration tests
- **Documentation**: Use inline documentation and README files
- **Tooling**: Modern development tools and build systems

### Portfolio Enhancement Priorities

1. **Code Quality**: Exemplary code that showcases expertise
2. **Documentation**: Clear, comprehensive, professional
3. **Innovation**: Demonstrate creative problem-solving
4. **Practical Value**: Real-world applicability
5. **Technical Depth**: Show deep understanding of technologies

### Special Considerations

- **Public Repository**: All code is publicly visible - maintain professional standards
- **Career Impact**: This portfolio may be reviewed by potential employers
- **Learning Value**: Projects should demonstrate growth and learning
- **Originality**: Showcase unique approaches and original thinking
